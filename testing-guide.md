# JarvisAgent 测试指南

本文档描述如何测试 JarvisAgent 的所有功能场景，以及 2026年5月5日 新增的功能说明。

---

## 环境准备

### 1. 启动服务器

```bash
cd server
pip install -r requirements.txt
python main.py
```

服务器默认启动在 `http://localhost:8000`

### 2. 打开前端

浏览器访问 `http://localhost:8000`

### 3. 多端测试

打开多个浏览器窗口/标签页（或使用不同浏览器），每个窗口登录不同账户即可模拟多端通信。

---

## 新增功能（2026-05-05）

| 功能模块 | 说明 |
|---------|------|
| 用户注册/登录 | JWT 认证，支持注册新账号和密码登录 |
| 好友系统 | 搜索用户、发送好友请求、接受/拒绝、自动创建私聊 |
| 多设备 WebSocket | 同一用户多个标签页同时在线，消息同步推送 |

| 演示稿编辑 (edit_slides) | 通过 Agent 指令修改已有 PPT |
| 分享/交付 (share_deliverable) | 分享文档或演示稿的链接 |
| Planner 增强 | 识别更多意图（编辑、分享、总结等） |

## 场景 A：IM 入口与消息发送

**目标**：验证基本的即时通讯功能。

### 测试步骤

1. **注册账号**：打开页面 → 点击"注册新账号" → 填写用户名、显示名称、密码 → 点击注册
2. **登录**：输入已注册的用户名和密码 → 点击登录
3. **查看会话列表**：左侧应显示 "JarvisAgent" 的默认会话
4. **发送普通消息**：点击 JarvisAgent 会话 → 在输入框输入"你好" → 发送
5. **收到回复**：Agent 应返回普通对话回复（general_reply）

### 预期结果

- 注册成功后自动登录，进入主界面
- 会话列表显示 Agent 会话和欢迎消息
- 普通聊天触发 general_reply 工具，返回友好回复

## 场景 B：任务规划（Planner）

**目标**：验证 Agent 能正确理解用户意图并生成执行计划。

### 测试步骤

1. 在 Agent 会话中输入："帮我写一份产品需求文档"
2. 观察返回的消息（应包含任务进度卡片）

### 意图识别测试

| 输入 | 期望识别的工具 |
|------|-------------|
| "帮我写一份产品方案" | create_document |
| "做一个10页的PPT" | create_slides |
| "总结一下我们的讨论" | summarize_chat |
| "你好，你能做什么？" | general_reply |
| "修改一下文档的标题" | edit_document |
| "把PPT第三页的内容改一下" | edit_slides |
| "把文档分享给大家" | share_deliverable |

### 预期结果

- 消息发送后，收到 Agent 状态卡片（正在规划 → 正在执行 → 完成）
- 不同指令触发对应的工具

## 场景 C：文档生成与编辑

**目标**：验证文档的创建和编辑功能。

### 创建文档测试

1. 在 Agent 会话输入："帮我写一份AI产品市场分析报告"
2. 等待 Agent 执行完毕
3. 点击导航栏"📄 文档"标签
4. 左侧列表应出现新生成的文档
5. 点击文档查看内容

### 编辑文档测试

1. 先创建一个文档（步骤如上）
2. 回到 Agent 会话，输入："修改文档，添加一个关于竞品分析的章节"
3. 再次查看文档，确认内容已更新

### 预期结果

- 文档创建后出现在文档列表中
- 文档内容包含标题、章节、正文
- 编辑后文档内容有对应变化
- Agent 返回的卡片中有"查看文档"按钮

## 场景 E：多设备同步

**目标**：验证同一用户多设备登录时消息同步。

### 测试步骤

1. **窗口1**：打开浏览器标签页A → 登录用户 "testuser"
2. **窗口2**：打开浏览器标签页B → 使用相同账号 "testuser" 登录
3. **窗口1**：在 Agent 会话中发送消息 "帮我写一份周报"
4. **窗口2**：观察是否同步收到消息和 Agent 回复

### 多用户测试

1. **窗口1**：登录用户 A
2. **窗口2**：登录用户 B
3. A 和 B 互为好友后（参见场景 F 的好友添加步骤）
4. 在 A 的私聊窗口发消息给 B
5. B 应实时收到消息

### 预期结果

- 同一用户多设备登录，消息同步推送到所有设备
- 不同用户间可通过好友私聊实时通信
- WebSocket 断线重连后不丢失消息

## 好友系统测试

**目标**：验证好友搜索、添加、接受/拒绝功能。

### 测试步骤

1. **准备**：在两个窗口分别注册/登录不同账号（如 user1 和 user2）
2. **搜索好友**：
   - user1 点击"👥 好友"标签
   - 在"添加好友"搜索框输入 "user2" → 点击搜索
   - 在搜索结果中点击"添加好友"
3. **接受请求**：
   - user2 点击"👥 好友"标签
   - 在"好友请求"区域看到 user1 的请求
   - 点击"接受"
4. **验证私聊**：
   - 双方会话列表中应自动出现对方的私聊会话
   - 可以在私聊中互发消息

## 常见问题

### Q: Agent 没有回复？
确认 `.env` 文件中配置了 `ANTHROPIC_API_KEY`。如果没有 API Key，系统会使用 fallback 模式（关键词匹配 + 模板生成），功能依然可用但生成质量较低。

### Q: WebSocket 连接失败？
检查浏览器控制台错误信息。确保服务器正在运行且端口未被占用。

### Q: 好友请求没有实时通知？
确认双方都已通过 WebSocket 连接（登录后自动连接）。可以刷新页面重新建立连接。

### Q: 种子用户登录失败？
种子用户（alice, bob 等）第一次登录时任意密码即可（系统会自动设置该密码），之后需使用设置的密码登录。
